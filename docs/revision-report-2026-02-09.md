## ✅ AIガバナンス記事：Before → After（冒頭3段落の要点）

### 冒頭リード（全記事共通パターン）

**Before**
> ［タイトル］は、企業のAIガバナンスにおいて経営・法務・監査が意思決定するうえで重要なテーマです。規制・フレームワークの動向をふまえ、実装と証跡の両面で使える形に整理します。本稿では背景、設計のポイント、監査・説明責任への接続まで一通り押さえ、次のアクションに繋げられるようにします。

**After**
> ［タイトル］について、実務に落とすうえで「何を証跡として残し、誰が責任を負うか」を明確にすることが、監査と説明責任の前提になる。本稿では背景・設計・監査接続の順に整理する。

### 設計のポイント段落

**Before**
> ポリシーとプロセスを分けて設計し、プロセスでは申請・承認・記録・例外管理を一気通貫で回せるようにします。ツールで自動化できる部分と、人の判断が必要な部分を切り分け、証跡が残る設計にします。

**After**
> ポリシーと手続きを分け、申請・承認・記録・例外管理を一つの流れで扱えるよう設計します。自動化する部分と人の判断が必要な部分を切り分け、証跡が残る形にします。

### CTAボックス

**Before**
> 利用規程、評価・透明性の設計から監査証跡まで、一気通貫でサポートします。

**After**
> 利用規程の設計から評価・透明性・監査証跡まで、範囲を決めたうえで支援します。

## 品質ゲート自己チェック結果

### 全ページ共通
- **固有の一文** … サービス「AIガバナンス・リスク管理」に「RACIが曖昧で承認が回らない・例外申請の証跡が残らず監査で指摘される」を追加。他ページは共通テンプレ削減により「どこでも言える一般論」を削減。
- **抽象スローガンで終わらない** … 冒頭リードを「実務の問い」型に変更。CTAを「支援」「範囲を決めたうえで」に変更。
- **箇条書き比率** … 構造は維持しつつ、本文の「一気通貫」「〜します」連打を削減。
- **断定（必ず／完全／保証）** … 「保証」「コンプライアンス達成」「準拠」の断定を削除・条件付き表現に変更。
- **日本語見出しの英語** … Services / Company / Key Point / Contact Us 等を日本語に統一（固有名詞・製品名は除外）。

### AIガバナンス記事
- **規制・標準の準拠断定** … 「対応するガバナンス体制」「コンプライアンス強化」を「対応関係の整理」「不足箇所と対応方針の明確化」に変更。
- **監査可能性** … リードで「証跡・責任の明確化」を前面に。solutions で「継続評価・監査接続」を明示。
- **出典のない統計** … ブログ本文での無出典の罰金・億円表記は、今回の対象ファイルでは追加削除していない（必要なら別途対応）。
- **参考資料** … 既存の権威ソース（3つ以上）を維持。

## 今後の推奨対応

1. **blog/ai/** … 「できます」「可能に」「最適化」等が残る記事があるため、必要に応じて文脈に合わせた手直しを推奨。
2. **数値・統計** … blog/ai の「10年・1,000億円」「数億円規模」等は、出典明記または定性的表現への変更を推奨。
3. **リードの表記** … タイトルに「実務」が含まれる記事では「［タイトル］について、実務に落とすうえで」となり、「実務を実務に」の重複感がある場合あり。必要なら「〜の実務を、現場の手続きに落とすうえで」等に個別調整可能。
4. **英語サイト（en/）** … 本改稿では日本語サイトを対象。en/ は未対応。
